# Loan Prediction ML API

## 📌 Project Overview

This project predicts whether a loan will be **Approved** or **Rejected** using Machine Learning.

It is built using:

* Python
* Scikit-learn (RandomForestClassifier)
* FastAPI

## ⚙️ Features

* Data preprocessing & cleaning (missing value imputation)
* Feature engineering (`TotalIncome`, `EMI`)
* Handling class imbalance via `class_weight`
* Model training using RandomForest
* REST API using FastAPI

## 📁 Project Structure

```
loan-prediction-ml-ap/
├── app.py              # FastAPI app that serves predictions
├── train_model.py       # Data cleaning, feature engineering, training, evaluation, saving
├── train.csv             # Training dataset
├── loan_model.pkl        # Trained RandomForestClassifier (generated by train_model.py)
├── scaler.pkl             # Fitted StandardScaler (generated by train_model.py)
├── requirements.txt 
└── README.md
```

## 📊 Input Features

| Field | Type | Notes |
|---|---|---|
| Gender | "Male" / "Female" | |
| Married | "Yes" / "No" | |
| Dependents | "0" / "1" / "2" / "3+" | |
| Education | "Graduate" / "Not Graduate" | |
| Self_Employed | "Yes" / "No" | |
| ApplicantIncome | number | used to derive `TotalIncome` |
| CoapplicantIncome | number | used to derive `TotalIncome` |
| LoanAmount | number | also used to derive `EMI` |
| Loan_Amount_Term | number | also used to derive `EMI` |
| Credit_History | 0 / 1 | |
| Property_Area | "Rural" / "Semiurban" / "Urban" | |

`TotalIncome` and `EMI` are computed automatically inside `app.py` — you do not send them yourself.

## 🚀 How to Run

1. Install dependencies
   ```bash
   pip install -r requirements.txt
   ```

2. (Optional) Retrain the model — `loan_model.pkl` and `scaler.pkl` are already included,
   but you can regenerate them at any time:
   ```bash
   python train_model.py
   ```

3. Run the API
   ```bash
   uvicorn app:app --reload
   ```

4. Open Swagger UI
   ```
   http://127.0.0.1:8000/docs
   ```

## 🧠 Model Details

* Algorithm: Random Forest (`n_estimators=200`, `max_depth=5`, `min_samples_split=5`)
* Class imbalance handled using `class_weight={0: 4, 1: 1}` (0 = Rejected is the minority class)
* Feature engineering: `TotalIncome = ApplicantIncome + CoapplicantIncome`, `EMI = LoanAmount / Loan_Amount_Term`
* Test accuracy: ~0.92 on a held-out 20% split (recall on the "Rejected" class is noticeably weaker than on "Approved" — run `train_model.py` to see the full classification report)
